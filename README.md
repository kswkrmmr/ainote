# あいのて

## 1. サービス概要

* **アプリ名**：あいのて（合いの手 / 愛の手 / AIの手）
* このサービスは、伝えたい気持ちはあるのに言葉ですれ違ってしまう人が、相手と落ち着いて対話できるようになるサービスです。
* AIを第三者的な対話支援役として、感情的にならずに建設的な話し合いができる場を提供します。

---

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

* 家事や育児などについて夫婦間で話し合うとき、どちらかまたは両方が感情的になってしまう場面を自分自身が経験していた
* 感情的になることで「相手の話を聞けない」「自分の主張ばかりになる」「話し合い自体をやめてしまう」という状況が起きていた
* テレビで夫婦問題のカウンセリングを見て、「第三者がいると冷静になれる」ことに着目した
* 身近な人からも同様の話をよく聞く。夫婦間カウンセリングという仕事が存在することからも、多くの人が抱えている問題だと感じた

### 2-2. なぜそれが「気になった」のか

* 自分自身の実体験として感じていた課題であること
* 同じ悩みを周囲からもよく聞き、普遍的な問題だと感じたこと
* 夫婦間カウンセリングという仕事が成立していることからも、需要があると判断できること

---

## 3. 課題の整理

### 3-1. 表に見えている困りごと

* 家事・育児について話し合っているときに感情的になってしまう
* 攻撃的な言葉になったり、意図しない意味で言葉が受け取られたりする
* 本来解決したい問題よりも、言葉のぶつかり合いが中心になってしまう

### 3-2. 本当に解決したい課題は何か

感情的な言葉によって、本来伝えたい気持ちが相手に届かなくなってしまうこと。

話し合いの場では、問題そのものよりも伝え方が原因で対立が深まることがある。その結果、「話しても無駄」という諦めにつながり、本来解決できたはずの問題まで放置されてしまう。

あいのてでは、AIが言葉を整理し、相手に伝わりやすい表現へ変換することで、対話そのものを支援する。

---

## 4. 想定ユーザーについて

### 4-1. 想定しているユーザー

#### 主な想定ユーザー：子育て中の夫婦

* 子供が小さい時期は、睡眠不足・育休・仕事復帰など夫婦のストレスが溜まりやすい
* 家事・育児・家計など話し合うべき問題が多い
* 年齢ではなく「子育て中」というライフステージに着目

#### 利用対象

夫婦に限らず、

* 恋人
* 親子
* 友人
* 職場

など対人関係全般で利用可能

#### 利用シーン

ストレスが爆発した後ではなく、話し合いが必要だと感じたタイミングで利用する。

### 4-2. 自分とユーザーの距離

* 自分自身
* 子育て中の父親として実際に感じている課題

### 4-3. 実際に使われる可能性について

* 最初に使ってくれそうな人：子育て中の夫婦や知人
* 現在の代替手段：我慢する、言い合いになる、カウンセリングを利用する
* 本サービスを使う理由：気軽に、すぐに、低コストで利用できる

### 4-4. ユーザーがサービスを利用するまでの流れ

* 話し合いが必要な状況になる
* アプリを開く
* パートナーを招待する
* 議題（Topic）を作成する
* AIを介して対話を行う
* 必要に応じて会話の整理を行う

## 5. 既存サービス・競合調査

### 5-1. 似たサービス

#### Ringi (https://app.ringi.ai/aiassistant)


カップル・夫婦向けAIアプリ。
AIアシスタントへ相談し、アドバイスを受ける形式。

### 5-2. それでも自分が作りたい理由

* RingiはAIに相談し、AIからアドバイスを受けるサービスである。
* あいのては当事者同士の対話を支援するサービスである。AIは感情的な言葉を相手に伝わりやすい表現に変換する通訳役として機能する。


### 5-3. 差別化ポイント

AIがいることで、2人が自分たちで答えを出せる。

感情的な言葉をAIが受け取り、棘の少ない表現へ変換して相手に届けることで、伝えたい気持ちを正しく伝えられる。

## 8. MVPで実現する機能

* メールアドレス・パスワードによるユーザー認証
* 対話ルーム作成
* ルーム招待機能
* 議題（Topic）作成
* AIによる文章整理・翻訳チャット

### MVPでのAI翻訳イメージ

入力：

「なんで私ばっかり家事してるの？」

AI翻訳後：

「家事の負担が偏っているように感じています。一緒に分担を見直せないでしょうか？」

---

## 9. 本リリースで実現したい機能

### AI翻訳後の編集機能

AIが生成した文章を参考にしながら、自分の言葉で修正して送信できる。

### 会話の整理機能

複数回のやり取りをもとに、

* Aさんの考え
* Bさんの考え
* 共通点
* 未解決の論点

を可視化する。

例：

Aさんの考え
・家事負担が偏っていると感じている

Bさんの考え
・仕事が忙しく時間が取れない

共通点
・家庭を大切にしたい

未解決の論点
・平日の家事分担

### LINE連携

* LINEによる招待共有（招待URLをLINEで簡単に送信できる）
* LINEログイン（LINEアカウントでログインできる）
* LINE通知（新しいメッセージが届いた際に、あいのて公式LINEアカウントから通知する）

## 10. 懸念点と対策

### 懸念点①：片方が利用したがらない

対策：

* シンプルなUI
* 招待しやすい導線
* 登録ハードルの低減

### 懸念点②：AI翻訳が意図と異なる

対策：

* AI翻訳後の編集機能を実装
* ユーザーが最終確認して送信する

### 懸念点③：AIが感情を弱めすぎる

対策：

* 感情を消すのではなく、伝わりやすい形に整理するプロンプト設計
* 元の意図を保持することを重視する

## 11. 技術スタック

| カテゴリ | 技術 | 選定理由 |
| --- | --- | --- |
| フロントエンド | React | チャット画面はメッセージ・入力欄・AI翻訳結果など状態管理が複雑になる。コンポーネント分割と状態管理が得意なReactがこの構成に適している |
| バックエンド | Rails API | ユーザー認証・ルーム管理・メッセージ保存など、定型的なCRUD処理が多い。Railsの規約ベースの開発でAPIを素早く構築できる。画面はReactに任せるためAPIモードで十分 |
| データベース | PostgreSQL | ユーザー・ルーム・メッセージの関係性をリレーショナルに管理する必要がある。Railsとの相性も良く、Renderでの管理もしやすい |
| デプロイ | Render | フロント・バックエンド・DBを一元管理でき、個人開発でのインフラ管理コストを最小化できる |
| AI | OpenAI API（GPT-4o mini） | このアプリのコア機能である「感情的な文章を穏やかな表現に変換する」処理に必要。MVPの検証フェーズではコストを抑えつつ十分な品質が得られる。品質に問題が出た場合はGPT-4oへの切り替えも容易 |
| スタイリング | Tailwind CSS | 感情的な状態で使うアプリのため、UIは穏やかで直感的である必要がある。細かいデザイン調整をクラスで素早く実装できる |
| UIコンポーネント | shadcn/ui | チャット画面・モーダル・ボタンなど必要なコンポーネントが揃っており、アクセシビリティも考慮済み。UX品質を保ちながら開発速度を上げられる |
| 環境 | Docker | React・Rails・PostgreSQLの複数サービスを組み合わせるため、環境差異によるトラブルを防ぐ必要がある |
